# Shoppingday 개발 지침

이 저장소에서 분석, 설계, 구현, 리뷰를 시작하기 전에 반드시
[`docs/product-strategy.md`](docs/product-strategy.md)를 먼저 읽고 프로젝트의 운영 전략과
MVP 우선순위를 이해한다.

키워드 후보, 상품명, 검색 노출 품질과 관련된 작업을 시작하기 전에는 추가로
[`docs/naver-search-quality-guidelines.md`](docs/naver-search-quality-guidelines.md)를 읽는다.
이 문서는 네이버 검색어드바이저의 웹 검색 품질 원칙을 Shoppingday에 적용하는 범위와
적용하지 말아야 할 범위를 구분한다.

스마트스토어 순위, 키워드 진입 전략 또는 적합도 필드 배치를 변경할 때는
[`docs/smartstore-ranking-operating-hypotheses.md`](docs/smartstore-ranking-operating-hypotheses.md)를
추가로 읽는다. 이 문서의 값은 사용자 운영 가설이며 실제 검색량·순위와 구분한다.

## 필수 제품 판단 기준

모든 기능과 기술적 선택은 아래 질문을 기준으로 평가한다.

> 이 기능이 1,000개의 제한된 상품 슬롯을 더 효율적으로 운영하고,
> 판매 가능성이 있는 상품의 매출을 높이는 데 도움이 되는가?

- 빠른 등록 단계에서는 처리 속도, 적정 품질, 낮은 AI 비용을 우선한다.
- 성장 관리 단계에서는 실제 반응이 확인된 상품에만 분석·추적·최적화 비용을 집중한다.
- 두 단계를 혼동하지 말고, 대량등록 경로에 불필요한 고비용 분석을 강제하지 않는다.
- 초기 MVP에 직접 도움이 되지 않는 자동화와 복잡성은 추가하지 않는다.
- 순위, 검색량, 매출 같은 사실 데이터는 추정값을 실제 값처럼 취급하지 않는다.
- 기능 범위나 우선순위가 불명확하면 `docs/product-strategy.md`의 목표에 더 직접적으로
  기여하는 쪽을 우선하고, 중대한 제품 방향 변경은 사용자와 확인한다.
