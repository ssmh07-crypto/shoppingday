# 프로젝트 배경과 핵심 목표

이 문서는 Shoppingday의 제품 방향을 결정하는 기준 문서다. 기능을 설계하거나 구현할
때는 기술적 편의보다 이 문서의 운영 목표를 먼저 고려한다.

## 배경

사용자는 과거 스마트스토어에 약 10,000~20,000개의 상품을 등록하던 대량등록
판매자였다. 당시에는 최대한 많은 상품을 빠르게 등록하고, 그중 일부에서 판매가
발생하도록 만드는 전략으로 전체 매출을 확보할 수 있었다. 따라서 개별 상품의
상품명·키워드·태그를 정교하게 최적화하는 일의 상대적 중요도는 낮았다.

스마트스토어 정책 변경으로 일정 매출 기준을 충족하지 못하면 등록 가능한 상품 수가
약 1,000개로 제한된다. 이제는 상품 수를 무한히 늘리는 전략보다 제한된 슬롯을
효율적으로 사용하고, 판매 가능성이 확인된 상품의 성과를 높이는 전략이 중요하다.

## 핵심 목표

서비스는 다음 두 가지 운영 단계를 명확히 지원해야 한다.

상품 등록 전의 **소싱 조사**는 위 두 단계에 들어가기 전의 의사결정 보조 흐름이다.
키워드 시장, 국내 가격, 고객 불편, 리뷰와 샘플을 기록해 제한된 상품 슬롯에 넣을
후보를 더 신중하게 고른다. 사용자가 직접 확인한 사실과 판단을 보존하되 특정 체크
조건만으로 재고 소진, 노출 또는 매출을 보장하지 않는다.

### 1. 빠른 등록 단계

- 처음 약 1,000개의 상품을 대량등록 방식으로 빠르게 확보한다.
- AI를 이용해 상품명, 키워드, 태그를 빠르게 괜찮은 수준으로 생성한다.
- 개별 상품에 과도한 분석 시간이나 AI 비용을 사용하지 않는다.
- 완벽한 최적화보다 처리 속도와 안정적인 기본 품질을 우선한다.

### 2. 성장 관리 단계

- 등록 후 실제 판매가 발생하거나 판매 가능성이 보이는 상품만 선별한다.
- 선별한 상품에 키워드 순위 추적, 키워드 변경, 상품명 수정 등의 관리 비용을
  집중한다.
- 키워드 및 상품명 변경 이력을 보존한다.
- 매출과 순위 변화를 함께 비교할 수 있도록 발전시킨다.
- 제한된 상품 슬롯 안에서 성장 상품의 매출을 최대화한다.

## 제품 판단 원칙

모든 기능은 아래 질문을 기준으로 설계한다.

> 이 기능이 1,000개의 제한된 상품 슬롯을 더 효율적으로 운영하고,
> 판매 가능성이 있는 상품의 매출을 높이는 데 도움이 되는가?

이 질문에 직접 도움이 되지 않는 복잡한 기능은 초기 MVP에서 제외한다.

추가 원칙은 다음과 같다.

- 빠른 등록과 성장 관리는 목적과 비용 구조가 다르므로 별도 흐름으로 설계한다.
- 모든 상품에 고비용 AI 분석이나 순위 추적을 자동 적용하지 않는다.
- 판매 반응이 확인된 상품에만 추가 비용과 운영 시간을 집중할 수 있어야 한다.
- AI 추천은 사용자의 판단을 돕는 도구이며, 사실 데이터나 최종 결정을 대신하지 않는다.
- 검색량, 순위, 매출 등은 신뢰할 수 있는 실제 데이터 출처를 사용하고 추정값과
  명확히 구분한다.
- 자동화는 상품 슬롯 효율과 매출 개선 효과가 명확할 때 단계적으로 추가한다.

## 현재 기능 설계에의 적용

- 소싱 조사 기능은 등록 전 후보의 키워드 시장·가격·리뷰·1688 샘플 근거를 한곳에
  기록하는 경량 조사 도구다. 검색수와 매출은 사용자가 확인한 값을 입력하며 추정값을
  자동 생성하지 않는다.
- 저장한 소싱 아이템은 상품 등록관리의 대기 항목으로 이어진다. 등록 준비를 시작하면
  별도 등록 모델을 중복 생성하지 않고 기존 상품 초안을 만들어 상품명·태그·가격을
  넘기며, 사용자가 카테고리·속성·재고·이미지·판매 정책을 확인한 뒤 발행한다.
- 기존 상품 등록 기능은 **빠른 등록 단계**에 해당한다.
- 스마트스토어 링크로 상품을 추가하고 키워드·상품명·변경 이력을 관리하는 기능은
  **성장 관리 단계**에 해당한다.
- 키워드 순위 추적은 성장 관리의 중요 기능이지만, 별도 지시가 있을 때 구현한다.
- 초기 키워드 관리 MVP는 상품 연결, 규칙 기반 상품 분석, 네이버 연관 키워드와 실제
  검색량 확인, 사용자 키워드 선택, 상품명 초안 편집 및 결과 저장에 집중한다.
- 성장 상품 키워드 관리는 외부 생성형 AI 선택 경로 없이 규칙 기반 기본 모드로
  동작한다. 검색량과 경쟁도는 네이버 검색광고 API의 실제 응답만 사용한다.

## 문서 유지 원칙

제품의 핵심 운영 전략이 바뀌면 이 문서를 먼저 갱신하고, 그 변화가 라우팅, 데이터
모델, 비용 정책, 외부 API 사용 범위에 미치는 영향을 함께 검토한다. 세부 구현 문서는
이 문서와 충돌해서는 안 된다.
