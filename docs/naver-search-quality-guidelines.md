# 네이버 검색 품질 가이드 적용 기준

마지막 확인: 2026-07-24

이 문서는 네이버 검색어드바이저 공식 가이드를 Shoppingday의 키워드 후보와 상품명
초안에 일관되게 적용하기 위한 내부 기준이다. 키워드·상품명·검색 노출 품질을 변경할
때는 `docs/product-strategy.md`를 먼저 읽고 이 문서를 함께 참고한다.

## 적용 범위와 주의점

네이버 검색어드바이저는 기본적으로 **네이버 웹 검색과 웹사이트 SEO**를 설명한다.
스마트스토어 쇼핑 검색의 노출 순위 공식이나 상품명 작성 규정 자체가 아니다.
따라서 아래 원칙은 상품명 품질을 높이는 참고 기준으로 사용하되, 특정 키워드나 제목이
쇼핑 검색 순위를 올린다고 보장하거나 실제 순위·검색량처럼 표시하지 않는다.

- 네이버 공식 가이드는 제목의 고정 글자 수 제한을 제시하지 않는다. 소싱 등록의
  40자 검토 안내와 50자 입력 제한은 Shoppingday 내부 운영 기준이며 네이버 공식
  제한으로 표시하지 않는다.
- 구조화 데이터, robots.txt, 사이트맵, canonical은 자체 웹사이트를 운영할 때 검토할
  항목이다. 현재 스마트스토어 상품명 생성 로직에는 적용하지 않는다.
- 구조화 데이터 적용도 검색 노출을 보장하지 않는다.
- 실제 검색량은 네이버 검색광고 API 응답만 사실 데이터로 표시한다.

직접 참고하는 일반 품질 원칙은 사용자 중심 정보, 정확하고 고유한 제목, 불필요한
키워드 남용과 중복 표현 방지, 콘텐츠 주제와 제목의 일치, 상위 노출을 보장하지 않는
표현이다.

### 공식 가이드 항목별 적용 판단

| 공식 가이드 내용 | Shoppingday 상품명 적용 |
| --- | --- |
| 사용자가 콘텐츠의 핵심을 바로 파악할 수 있는 정확하고 간결한 제목 | 구체 상품 유형을 반드시 남기고 수식어를 제한적으로 조합한다. |
| 콘텐츠와 무관한 인기·화제 키워드 금지 | 사용자가 상품명으로 분류했더라도 기본 상품 유형과 연결되지 않는 키워드는 자동 조합에서 제외한다. |
| 동일 키워드 반복과 단순 키워드 나열 방지 | 정규화 중복과 동일 상품 유형의 반복은 제거한다. 실제 후보에 있는 서로 다른 동의 상품형은 읽기 기준점으로 제한적으로 사용할 수 있다. |
| 스팸성 키워드와 홍보 문구 나열 금지 | 무료배송·특가·할인 등 홍보성 표현을 규칙 초안에서 제거한다. |
| 페이지마다 주제에 맞는 고유한 제목 | 초안은 각 소싱 아이템의 분류 결과로 생성한다. 다른 상품과의 동일 제목 검사는 별도 상품 목록 검사가 필요하므로 현재 자동 보장하지 않는다. |
| 제목을 과도하게 길게 작성하지 않기 | 소싱 등록은 40자 초과 시 내부 검토 안내를 하고 50자를 강제 상한으로 둔다. 이는 공식 고정 제한이 아니라 빠른 등록 품질을 위한 내부 운영 기준이다. |
| 제목을 검색 노출만을 목적으로 자주 변경하지 않기 | 빠른 등록 초안은 사용자가 확정하고, 등록 후 변경·반응 비교는 성장 관리 이력에서 다룬다. |
| 허위·과장·낚시성 표현 및 실제 내용과 다른 제목 금지 | 검색량이 높다는 이유만으로 다른 품목명으로 바꾸지 않고 사용자가 최종 확인한다. |

`robots.txt`, 사이트맵, canonical, URL, 리다이렉트, HTML의 `title`·`H1`·`meta
description`, Open Graph, 이미지 `alt`, 링크·백링크, 구조화 데이터, 수집·색인과
웹사이트 보안 항목은 공식 가이드의 중요한 부분이지만 스마트스토어 상품명 문자열을
조합하는 규칙은 아니다. Shoppingday 자체 웹사이트의 SEO를 작업할 때 별도로 적용한다.

## Shoppingday에 적용할 원칙

### 1. 정확하고 간결한 상품명

- 제목만 보고 실제 상품을 구분할 수 있어야 한다.
- 구체적인 상품 유형은 유지한다. 예: `골무`.
- 같은 단어를 반복하지 않는다.
- 구체적인 상품 유형이 있으면 `부자재`, `용품`, `상품`, `제품`, `세트`, `도구`처럼
  넓은 분류어는 초안에서 우선 제외한다.
- 소싱 등록에서는 40자를 넘으면 간결하게 다듬으라는 안내를 표시하고 50자를 넘는
  입력과 추천 결과는 허용하지 않는다.

### 2. 관련 없는 인기 검색어와 모순 속성 차단

- 검색량이 높아도 상품과 직접 관련 없는 단어는 후보나 상품명에 넣지 않는다.
- 소재, 색상, 대상처럼 상품 입력값과 충돌하는 연관 검색어는 제외한다.
- 검색량이 많다는 이유만으로 실제 상품명을 다른 인기 품목명으로 바꾸지 않는다.

### 3. 홍보성·스팸성 표현 제외

`무료배송`, `정품`, `신상품`, `인기`, `추천`, `특가`, `할인`, `이벤트`, `국내배송`,
`당일배송` 같은 홍보 문구는 규칙 기반 상품명 초안에서 제외한다. 상품의 객관적인
속성보다 클릭을 유도하는 문구를 우선하지 않는다.

### 4. 사람이 최종 결정

- 생성 결과는 수정 가능한 초안으로 제공한다.
- 품질 문제는 경고하되 단순 권장사항만으로 저장을 막지 않는다.
- 기존에 저장한 상품명을 자동으로 덮어쓰지 않는다.
- 상품명 변경 이력과 변경 후 실제 반응 비교는 성장 관리 단계에서 다룬다.

### 5. 1,000개 상품 슬롯 관점

- 빠른 등록 단계에는 규칙 기반 기본 모드로 중복·일반 분류어·홍보어를 제거한다.
- 고비용 분석은 실제 판매 반응이 확인된 상품에만 선택적으로 사용한다.
- 동일·유사 상품명 경고로 제한된 슬롯을 서로 구분되는 상품으로 운영한다.

## 현재 구현 규칙

상품명 초안은 다음 순서로 만든다.

1. `상품명`으로 직접 분류한 키워드 중 월 검색수 1,000 이하만 사용한다. 이 수치는
   네이버 공식 SEO 기준이 아니라 사용자가 정한 소싱 운영 기준이다.
2. 공백·대소문자·유니코드 표기를 정규화하고 동일 키워드를 한 번만 남긴다.
3. 소싱 키워드가 복합 키워드에 포함되면 소싱 키워드를 기본 상품 유형으로 삼는다.
   그렇지 않으면 가장 짧은 구체 키워드를 기본 상품 유형으로 삼는다.
4. 기본 상품 유형을 포함하는 키워드에서 수식어와 상품 유형을 분리한다. 동일한 상품
   유형은 반복하지 않지만, 실제 상품명 후보에 `욕실화`와 `욕실슬리퍼`처럼 명확한
   동의 상품형이 함께 있으면 두 명사형을 읽기 기준점으로 살릴 수 있다. 이때
   `낮은 욕실화 앞막힌 예쁜 안 미끄러운 국산 욕실슬리퍼`처럼 수식어만 길게 몰아놓지
   않고 `수식어 + 명사` 단위 사이에 나머지 특징을 배치한다. 기본 상품 유형과
   연결되지 않은 키워드는 단순 나열하지 않는다. 두 번째 명사형 후보가 여러 개면
   `국산 욕실슬리퍼`처럼 명사와 자연스럽게 결합하는 원산지 표현을 우선 기준점으로
   사용한다. 수식어가 많아 긴 제목이 되면 동일한 기본 명사는 최대 두 번까지만
   배치해 중간 읽기 기준점을 추가할 수 있다. 이는 네이버 공식 반복 횟수 규정이
   아니라 Shoppingday 내부 가독성 규칙이다.
5. `무료배송`, `정품`, `신상품`, `인기`, `추천`, `특가`, `할인`, `이벤트`,
   `국내배송`, `당일배송` 같은 홍보성 표현을 제거한다.
6. 구체적인 기본 상품 유형 대신 `부자재`, `용품`, `상품`, `제품`, `세트`, `도구`
   같은 넓은 분류어만 남는 경우 이를 제거한다.
7. 카테고리 키워드는 상품명에 넣지 않으며 사용자가 편집해 넣어도 상품 초안 반영
   단계에서 차단한다.
8. 태그 키워드는 검색수와 관계없이 후보 전체를 별도로 추출한다. 상품 등록 초안에서
   사용자가 30자 이하 후보 중 최대 20개를 선택한다.
9. 속성 키워드는 상품명에 넣지 않고 네이버 공식 카테고리 속성과 대조할 참고 목록으로
   유지한다.
10. 관련 상품명 후보 전체를 의미군별로 정리해 조합에 검토하며, 기본 상품 유형을
    남긴 상태에서 50자를 넘기게 하는 수식어는 제외한다.
11. 후보를 검색수 순서대로 그대로 나열하지 않는다. 낮은 높이·문 걸림, 앞막힘, 배수,
    빠른 건조, 미끄럼 방지, 착화감, 무게, 크기처럼 특징 의미군을 구분한다. 같은
    의미군에서는 사용자가 고른 기준 검색어, 기본 상품 유형과의 연결, 실제 월 검색수,
    구체성 순으로 대표 표현 하나를 고른다. 낮은 검색수의 긴 표현이 높은 검색수의
    관련 표현을 밀어내지 않으며 연결되지 않은 후보는 조합 전에 제외한다.
12. `문에안걸리는`은 `문에 안 걸리는`, `발등낮은`은 `발등 낮은`처럼 사람이 읽기
    어려운 결합 표현의 표시용 띄어쓰기를 보정한다.
13. 40자 초과, 제거된 홍보어·분류어, 연결되지 않은 키워드를 경고로 보여주며 사용자가
    수정 가능한 초안으로 제공한다. 소싱 등록 상품명은 입력 단계에서도 50자로 제한한다.
    관련 후보 전체에서 최우선 키워드를 유지하면서 50자를 넘지 않는 조합 중 제목
    길이를 가장 충분히 활용하는 조합을 선택한다.
14. 상품 등록 전용 편집에서는 검색수 1,000 이하 상품명 후보 중 하나를 기준 검색어로
    고를 수 있다. 선택한 후보의 수식어를 먼저 배치하되 나머지 품질 규칙은 동일하게
    적용한다.
15. 추천 결과에는 후보 전체가 아니라 최종 상품명에 실제 반영된 상품명 키워드만
    검색수와 함께 표시한다.

일반 상품 관리의 판매용 상품명 추천도 같은 품질 원칙을 사용하지만 소싱 분류 데이터는
사용하지 않는다. 현재 판매용 상품명과 선택된
네이버 카테고리에서 상품 유형·소재·재질·용도를 규칙으로 분리하고, 네이버 검색광고
키워드 도구는 관련 조합의 실제 월간 검색량을 확인하는 용도로만 사용한다. 연관
키워드가 상품 유형을 포함하지 않거나 입력 소재와 충돌하면 추천 근거에서 제외한다.
추천 결과는 미리보기 후 사용자가 적용하며 자동 저장하거나 네이버 상품에 자동
반영하지 않는다.

이 규칙은 검색순위 상승을 보장하는 최적화 공식이 아니라, 부정확하거나 반복적인
상품명을 줄이기 위한 품질 안전장치다.
