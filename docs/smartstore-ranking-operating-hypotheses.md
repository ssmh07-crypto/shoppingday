# 스마트스토어 순위 운영 가설

마지막 갱신: 2026-07-19

이 문서는 사용자가 실제 스마트스토어 운영에 적용할 전략을 Shoppingday가 일관되게
기억하기 위한 기준이다. 네이버가 공개한 고정 배점표나 노출 보장 공식이 아니라
`사용자 운영 가설`이다. 화면에서도 실제 검색량·실제 순위와 구분해 표시한다.

## 적용할 운영 가설

1. 순위 점수는 `인기도 × 적합도 × 신뢰도` 구조로 이해한다.
2. 적합도에 영향을 주는 필드의 힘은 `상품명 > 스토어명 > 카테고리 > 속성 > 태그`
   순서로 가정한다.
3. 자연검색 1페이지는 40개 상품으로 가정한다.
4. 같은 시각에 같은 키워드를 검색하면 사용자에게 동일한 순위가 보인다고 가정한다.
5. 월간 검색량 1,000 이하 키워드를 신규 판매자의 초기 공략 후보로 표시한다.
6. 소싱 등록 상품명은 검색수 1,000 이하의 관련 후보 전체를 의미군별로 정리해 조합에
   검토하고 최종 50자로 제한한다. 50자는 네이버 공식 배점이나 제한이 아니라 빠른
   검토를 위한 사용자 운영 기준이다.

## 구현 원칙

- 검색량 1,000 이하라는 이유만으로 상품과 무관한 키워드를 추천하지 않는다.
- 검색량 순으로 의미가 겹치는 키워드를 반복 배치하지 않고, 서로 다른 상품 특징을
  설명하는 후보를 우선한다.
- 인기도·적합도·신뢰도 입력값이 모두 실제로 확보된 경우에만 가설 점수를 계산한다.
  누락된 값을 추정해 채우지 않는다.
- `초기 공략 후보`, `운영 가설 점수`로 표시하며 네이버 공식 점수, 상위 노출 확률,
  1페이지 진입 보장이라고 표현하지 않는다.
- 상품명·스토어명·카테고리·속성·태그의 실제 등록값을 보존한 뒤 키워드가 어느
  필드에서 뒷받침되는지 비교한다.
- 검색 결과 표본을 기록할 때 기준 키워드, 확인 시각, 기기와 표본 수를 함께 저장한다.

## 추가로 필요한 사용자 데이터

- 스토어명
- 상품별 실제 네이버 카테고리
- 상품별 등록 속성 이름과 값
- 상품별 검색 태그
- 키워드별 실제 순위와 확인 시각
- 가능하면 클릭 수, 판매량, 구매평 수와 품절 기간

이 데이터가 없으면 검색량 기반 초기 공략 표시는 가능하지만 필드별 적합도 분석이나
인기도 점수 계산은 실행하지 않는다.
