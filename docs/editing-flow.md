# 상품 편집 흐름

1. 공급사 상품을 가져오면 원본은 `supplier_products`에 저장하고 판매 초안은 `products`에 한 번 생성한다.
2. 목록과 편집 화면은 DB만 조회한다. 친구도매 API를 호출하지 않는다.
3. 사용자는 원본 읽기 전용 영역을 참고해 판매 필드를 편집한다.
4. 임시저장은 Zod 검증, HTML 정화, `draft_version` 충돌 검사, 조건부 업데이트, 감사 로그 기록 순서로 수행한다.
5. 등록 준비 완료는 필수값을 검사해 내부 상태만 `ready`로 바꾼다.
6. ready의 중요 필드를 다시 저장하면 `editing`으로 돌아간다.
7. 이미지/옵션 원본 초기화는 확인 후 명시적으로만 수행한다.
8. 다음 단계에서는 `products`를 참조하는 `product_publications`와 스마트스토어 게시 서비스를 연결한다. 현재 게시 동작은 없다.

자동저장은 구현하지 않았다. 입력 변경 시 상단에 미저장 상태가 표시되고, 페이지 이탈 시 브라우저 경고가 나오며, 수동 임시저장 결과와 시간이 표시된다.
